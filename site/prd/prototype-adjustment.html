建图与导航工具 V1.0.0 第 3 类差异原型调整说明

来源：基于“两者都有，但口径 / 实现不一样”的已确认决策整理。
用途：给原型修改使用。
范围：只覆盖当前位置设点、地图选点设点、点位导航验证三类交互。

1. 使用机器人当前位置设点

PRD 口径：当前位置设点默认使用机器人当前位置和当前朝向。

原型调整：

  • 点位配置页保留“使用机器人当前位置设点”入口。
  • 用户点击后立即新增一个点位。
  • 新点位默认名称为 点位 N，编号为 P00N。
  • 点位坐标读取机器人当前位置。
  • 点位朝向读取机器人当前朝向。
  • 新增后自动选中该点位，并在右侧点位卡片中展开详情。
  • 地图上的点位标记需要显示朝向箭头。

不需要：

  • 不弹出单独确认弹窗。
  • 不要求用户二次选择朝向。
  • 不把当前位置设点做成长期高亮状态。

2. 地图选点设点

PRD 口径：点击生成点位，长按拖拽设置朝向。

原型调整：

  • 点位配置页提供“地图选点”模式。
  • 开启后，鼠标进入地图区域时显示选点光标。
  • 点击可通行区域后直接生成点位。
  • 点击没扫到、障碍物或地图外区域时不生成点位，并提示“只能放在可通行区域”。
  • 点位生成后自动选中，并展示默认名称和编号。
  • 用户长按点位标记并拖拽，地图上出现方向线和朝向预览。
  • 松开鼠标后写入该点位朝向。
  • 已保存或已创建的点位也支持长按拖拽调整朝向。

不需要：

  • 不把点位方向放到独立弹窗里设置。
  • 不要求用户输入角度数值作为主要交互。

3. 点位导航验证

PRD 口径：App 支持建图 + 导航；本期导航只做单点验证。

原型调整：

  • 点位卡片选中后，展示“导航到点位”按钮。
  • 点位没有坐标、没有朝向、在障碍物 / 没扫到 / 地图外时，按钮置灰并展示原因。
  • 点击“导航到点位”后进入验证中状态。
  • 验证中显示状态条：目标点位名称、前往中、停止按钮。
  • 验证中禁止编辑点位、拖动点位、调整朝向。
  • 用户点击停止后，提示“已停止前往点位”。
  • 遇到障碍物、手柄接管、定位丢失、急停、连接断开、无法规划路径时，停止本次验证并提示原因。
  • 到达点位且朝向对齐时，提示“已到达点位”。
  • 到达附近但朝向未对齐时，提示“已到达附近，朝向未对齐”。

不需要：

  • 不做多点路线。
  • 不做导览任务。
  • 不做导航策略选择。
  • 不做自动绕行后继续验证。
