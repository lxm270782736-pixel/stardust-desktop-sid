建图与导航工具 V1.0.0 项目需补充子需求清单

原 PRD：建图与导航工具 V1.0.0

1. Why

72 版本导览项目已有地图查看、点位和导航相关基础能力，但还缺少建图工具主流程必须闭环的能力。本文只整理“PRD 已确认、项目当前需要补充”的子需求，方便研发评估工期。

优先级口径：

  • 需补充P0：当前项目缺少，不补则建图工具主流程无法成立。
  • 需补充P1：当前项目缺少，不影响最小建图闭环，但影响现场效率或业务衔接。
  • 需补充P2：当前项目缺少，属于体验优化或低频补充。

2. What：项目需补充功能清单

S02 网页遥控 F04-06 / F04-07 / F04-08 用户在手柄不可用或需要备用时，用网页移动机器人。 不补则没有备用控制方式，现场建图容易中断。 需补充P0

3. How：P0 子需求说明

S01 边建图边显示

用户任务：用户遥控机器人移动时，能看到地图随扫描逐步生成。

页面：建图工作台。

显示内容：

  • P0：地图底图按建图数据逐步显示没扫到、可通行、障碍物。
  • P1：机器人位置和朝向、轨迹、扫描点。
  • P1：缩放、平移、适配视图、栅格显示开关。

规则：

  • 建图开始后，地图随建图数据更新。
  • 没有新数据时，地图保持当前结果，不自动伪增长。
  • 扫描点、轨迹和栅格只用于辅助判断，不作为保存后的地图分类。

异常处理：

  • 扫描数据中断：提示“扫描数据暂未更新”，保留当前地图。
  • 机器人定位异常：提示“机器人位置暂不可用”，地图继续保留。
  • 建图服务异常：停止地图更新，提示用户重试或返回地图列表。

验收口径：

  • 用户移动机器人后，地图底图能逐步变化。
  • 无数据时不会生成假地图。
  • 辅助图层缺失不阻断 P0，但需要按 P1 进入后续补充。

S02 网页遥控

用户任务：用户在手柄不可用或需要备用时，用网页控制台移动机器人完成建图。

页面：建图工作台；点位配置页如需使用当前位置设点，可复用同一套控制规则。

控制方式：

  • 按钮控制：前进、后退、左移、右移、左转、右转、停止。
  • 键盘控制：W / ↑ 前进，S / ↓ 后退，A 左移，D 右移，Q / ← 左转，E / → 右转，Space / Esc 停止。
  • 转向规则：单击转向按钮转 45°；长按连续转动，每秒 15°。

控制权规则：

  • 手柄有控制权时，网页按钮和键盘不生效。
  • 网页控制中收到手柄控制信号，网页控制立即停止。
  • 页面切换、窗口失焦、连接断开或控制台收起时，网页控制停止。

安全规则：

  • 用户松开按键或按钮后停止连续移动。
  • 距离障碍物过近时停止移动并提示。
  • 急停、底盘异常或连接异常时禁止发控制指令。

验收口径：

  • 按钮和键盘都能触发移动。
  • 手柄控制时网页控制不可抢占。
  • 异常或离开页面时不会持续发移动指令。

S03 下载地图

用户任务：二开用户下载地图包，用于开发、调试或导入自有系统。

入口：

  • 地图列表。
  • 地图查看页。
  • 业务 App 选图流程中可查看的地图详情入口。

下载内容：

  • 地图文件。
  • 点位列表。
  • 必要元数据：地图 ID、地图名称、保存时间、地图精度、文件状态。

规则：

  • 只有文件状态正常的地图可下载。
  • 无法读取的地图不可下载。
  • 下载动作不改变地图内容。

异常处理：

  • 下载失败：提示失败原因，允许重试。
  • 文件不存在或无法读取：提示地图文件异常，建议删除或重新建图。

验收口径：

  • 用户能从地图入口下载地图包。
  • 下载包能区分地图文件、点位列表和必要元数据。
  • 下载失败时页面不跳转，用户可重试。

S04 业务 App 选图返回

用户任务：自动回充 / 导览从建图工具选择地图，并拿到可继续使用的地图结果。

入口：

  • 自动回充 App 调起建图工具选图。
  • 导览 App 调起建图工具选图。
  • 已有素材库或业务入口需要选择地图时，进入同一选图流程。

流程：

  1. 来源 App 打开地图选择页。
  2. 有地图：用户选择一张可用地图并返回来源 App。
  3. 无地图：用户新建地图；保存成功后回到地图选择页。
  4. 用户取消：返回未选择结果。

返回内容：

  • 地图 ID。
  • 地图名称。
  • 文件状态。
  • 最近保存时间。
  • 点位数量。
  • 点位更新时间。

规则：

  • 无法读取的地图不可选择。
  • 来源 App 只接收选择结果，不在建图工具内配置回充、导览任务、路线或导航策略。
  • 新建地图来自业务 App 调用时，保存后保留来源上下文。

异常处理：

  • 来源上下文丢失：提示用户回到来源 App 重新选择。
  • 选中地图变为不可读：阻止返回，并提示选择其他地图或重新建图。

验收口径：

  • 自动回充和导览都能打开选图页。
  • 选中地图后能返回来源 App。
  • 无地图时能先建图，再回到选图流程。

4. P1 / P2 子需求处理口径

编号 功能 处理口径
S05 建图前选择栅格精度 按 P2 评估；若建图服务要求建图前传入精度，则提升到 P1。
S06 保存并编辑 按 P1 评估；不影响“保存后回列表”的最小闭环。
S07 保存失败保留并重试 按 P1 评估；建议与保存能力一起实现，避免现场成果丢失。
S08 独立地图查看页 按 P1 评估；若当前项目已有可复用查看页，优先复用。
S09 点位配置差异补齐 按 P1 评估；复用项目已有点位列表、地图标记、拖拽、编辑、删除和保存能力，只补齐地图选点朝向、点位字段口径、点位页移动控制台和合法性校验。
S10 点位导航验证 按 P1 评估；本期只做单点验证，不做多点路线、巡航或导航策略配置。
